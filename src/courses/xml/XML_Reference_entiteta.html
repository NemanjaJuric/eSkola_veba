Reference entiteta

Treba biti pažljiv prilikom korišćenja specijalnih karaktera jer neki karakteri imaju posebno značenje u XML standardu. Recimo, ukoliko bi se unutar tekstualnog sadržaja elementa ili unutar vrednosti atributa naveo samo karakter „<“, došlo bi do greške jer ga XML parser prepoznaje kao oznaku za početak novog elementa. Takođe, karakter „&“ se ne sme navesti kao takav jer je on rezervisan za oznaku početka reference entiteta.

Greška opisana primerom 12 se može ispraviti na način koji prikazuje primer 13.

Primer 12: Neispravan zapis poređenja <poredjenje> 700 < 1500 </poredjenje> Primer 13: Ispravan zapis poređenja <poredjenje> 700 &lt; 1500 </poredjenje> U pravilno napisanom primeru 13 je korišćena referenca entiteta „&lt;“ umesto specijalnog karaktera „<“. Reference entiteta imaju tri obavezna svoja dela, a to su ampersand (&), naziv entiteta i znak „tačka-zarez“ (;). Predefinisane reference entiteta u XML-u su:

  • &lt; za znak "<" (skraćeno od less than - manje od);
  • &gt; za znak ">" (skraćeno od greater than - više od);
  • &amp; za znak "&" (skraćeno od ampersand);
  • &apos; za znak "'" (skraćeno od apostrophe - apostrof);
  • &quot; za znak """ (skraćeno od quotation mark - navodnici);

Osim navedenih referenci mogu se koristiti i numeričke i heksadekadne numeričke reference entiteta. Recimo, znak „<“ osim što se može navesti pomoću reference „&lt;“, može se navesti i kao „&#60“ ili „&#x3C“. Svaku od ovih referenci će parser prilikom čitanja zameniti znakom „<“ i neće ih protumačiti kao početak etikete.

Samo karaktere „<“ i „&“ je striktno zabranjeno koristiti prilikom pisanja tekstualnog sadržaja elemenata i vrednosti atributa XML dokumenata, ali je dobra praksa koristiti reference entiteta i za ostale. Karakteri kao što su apostrof i navodnici koriste se za zadavanje vrednosti atributa, o čemu će reči biti kasnije.

Pomenute reference se smeju navoditi samo unutar tekstualnog sadržaja elemenata i vrednosti atributa. Ne smeju se pojaviti u imenima elemenata i atributa. Takođe, ukoliko se reference navedu u komentaru, njih parser neće zameniti odgovarajućim znakom, jer on prepoznaje reference samo unutar sadržaja elemenata ili vrednosti atributa.
