Definisanje vizuelnih komponenti

U svemu čime se programiranje bavi dobro je da stvari budu modularne, kompaktne i samostalne. ReactJS tu ideju primenjuje na kreiranje vizuelnih elemenata aplikacije. Osnovna ideja rada u ovom radnom okviru je da se vizuelni elementi podele na manje komponente, kako bi one bile ponovo iskoristive i kada se podaci koje prikazuju promene.

Definisanje vizuelnih elemenata u programskom jeziku JavaScript je prilično komplikovano i zahteva dosta poziva funkcije createElement. ReactJS omogućava da se vizuelne komponente definišu pomoću sintakse slične jeziku HTML, nazvane JSX (eng. JavaScript XML). Umesto pisanja koda za definisanje korisničkog okruženja, sintaksa JSX podrazumeva upotrebu odgovarajućih HTML oznaka.

Primer upotrebe JSX sintakse u radnom okviru ReactJS:

ReactDOM.render( <div> <h1> Naslov </h1> <h3> Podnaslov </h3> </div>, destination );

Drugačiji način kreiranja ReactJS okruženja:

ReactDOM.render(React.createElement "div", null, React.createElement( "h1", null, "Naslov" ), React.createElement( "h1", null, "Podnaslov" ) ), destination);

Sintaksa JSX omogućava jednostavno definisanje vizuelnih komponenti, a ipak zadržava moć i fleksibilnost koju nudi skript jezik JavaScript.

Napomena. Nije bitno da li je aplikacija napisana pomoću radnog okvira ReactJS ili neke druge biblioteke za JavaScript, krajni rezultat mora da bude kombinacija koda koja podrazumeva upotrebu jezika HTML, CSS i JavaScript, inače čitač veba neće znati šta da radi. Zato je potrebno definisati način na koji će se JSX konvertovati u odgovarajući JavaScript kôd, koji čitač veba razume.
