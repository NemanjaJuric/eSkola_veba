ES6 Sintaksa

Sintaksa je skup pravila koja definiše raspored simbola. Svaka grupa jezika ima svoju sintaksu. Sintaksa se odnosi na programske jezike u kojima dokument predstavlja izvorni kod i takođe se odnosi na označne jezike u kojima dokument opisuje podatke.

Program u okviru JavaScript-a sastoji se od:


Literali: Literal se može definisati kao oznaka za predstavljanje fiksne vrednosti u izvornom kodu. Generalno, literali se koriste za inicijalizaciju promenljivih. U sledećem primeru možete videti upotrebu literala u kojima 1 predstavlja celi broj literala, a niz "hello" je string literal. int x = 1; string str = "hello";
Variable: Promenljiva je lokacija skladišta koja je identifikovana pomoću memorijske adrese. Promenljiva je ime memorijskog bloka koji čuva vrednosti za program. Naziv varijable je standardni način referenciranja vrednosti.
Keywords: U kompijuterskom programiranju keyword je reč koja u posebnom kontekstu ima posebno značenje. Ne može se koristiti kao identifikator poput imena promenljive, funkcije i imena funkcije .
Operatori: Operatori su simboli koji definišu obradu operanda. Neki od uobičajenih primera operatora uključuju aritmetiku (dodavanje sa +), logičke operatore (poput AND ili &&) itd.


Komentari: Komentari pomažu programeru kada nešto želi da zabeleži ili naglasi

JavaScript podržava ova dva tipa komentara:

  • Multi-line comments: Ovi komentari se koriste za komentarisanje u više linija.
    /* Primer višerednog komentara */
  • Single-line comments:Ovi komentari se koriste za komentarisanje u jednoj liniji.
    //Jednolinijski komentar.

Identifikatori: Identifikatori su imena koja se daju elementima unutar programa, kao što su funkcije, promenljive itd. Postoje neka pravila za identifikatore koja su sledeća:

  • Identifikatori ne ukljucuju specijalne simbole sem znaka ($) ili donja crta(_).
  • Ime identifikatora ne sme biti ključna reč.
  • Identifikatori uključuju znakove i cifre. Ali ime identifikatora ne možete započeti cifrom.
  • Identifikatori ne smeju biti rastavljeni.

Na primer: U ovom primeru videćete pravilnu i nepravilnu deklaraciju identifikatora.

Pravilan primer: userName, user_name, name14, $name.

Nepravilan primer: Name@, user name, user-name, 14name.

Line Breaks i Whitespaces

ECMAScript ignoriše kartice, razmake i nove linije koje se pojavljuju u programima. Kartice, razmake i nove linije možemo lako koristiti u programu, a program slobodno formatirati i uvlačiti na pouzdan način, što povećava čitljivost koda i olakšava razumevanje..

JavaScript i Camel Case

JavaScript programeri bi trebali da koriste Lower Camel Case (koji počinje malim slovom).

e.g. userName, firstName, cityName, etc.

Treba napomenuti da crtice nisu dozvoljene u JavaScript-u, jer su crtice rezervisane za oduzimanje.

JavaScript is Case-Sensitive

Identifikatori razlikuju velika i mala slova.

Na primer:

username i userName su dve različite promenljive u JavaScript-u.

Upotreba tački sa zarezima nije obavezna u JavaScript-u. Ali ako u jednom redu ima više iskaza (niz uputstava), te izjave moraju biti odvojene tačkom i zarezom.

The Strict Mode

Strogi režim je uveden u ECMAScript 5 (peto izdanje specifikacije ECMAScript). Ovaj strogi mod možete koristiti u svim svojim programima. To će vam pomoći da napišete čisti kod, poput sprečavanja korišćenja neprijavljenih promenljivih.

Prednosti korišćenja striktong moda:

Postoji nekoliko promena koje strogi način rada čini normalnom JavaScript semantikom:

  • Strogi režim zabranjuje neke sintakse, što će verovatno biti definisano u budućim verzijama ECMAScript-a.
  • Strogi režim uklanja neke JavaScript tihe greške menjajući ih tako da bacaju greške.
  • Strogi režim takođe ispravlja greške, što otežava JavaScript motorima da izvršavaju optimizacije, tako da strogi režim ponekad čini brži rad u odnosu na identični režim.

Kako deklarisati strogi režim

Možete deklarisati dodavanjem "use strict"; na početku funkcije ili skripte.

Deklaracija početka skripte:Kada ga deklarišete na početku skripte, to će biti globalni domet, tj. Sav kod unutar skripte izvršit će se u strogom režimu .
Na primer:

"use strict"; example(); function example() { x= 89; // It will also cause an error because x is not declared<br> } Kada izvršimo ovaj primer, dobićemo grešku jer promenljivu nismo deklarisali x.

Deklaracija unut funkcije: Kada ga deklarirate unutar funkcije, to će biti lokalni domet, tj. Kod unutar funkcije bit će u strogom režimu.

y = 89; // It will not cause an error. example(); function example() { "use strict"; x = 89; // It will cause an error }
