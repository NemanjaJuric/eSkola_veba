Angular 4


Uvod

Angular 4 je radni okvir programskog jezika JavaScript. Koristi se za razvoj modernih veb aplikacija koje su zasnovane na dizajnu arhitekture koji se sastoji od veb komponenata međusobno povezanih pomoću njihovog posebnog U/I interfejsa.

Sastoji se od nekoliko biblioteka. Neke od njih čine osnovu radnog okvira, a neke su opcione.

Angular omogućava kreiranje reaktivnih jednostranih aplikacija (eng. Single Page Applications - SPA). Prednost korišćenja jednostranih aplikacija je, pre svega, njihova brzina odgovora koja je ekvivalentna brzini desktop aplikacija. Takođe, takve aplikacije mogu da rade i bez internet konekcije i imaju sposobnost samostalnog osvežavanja.

Prethodne verzije radnog okvira Angular su Angular 1 (poznat još i kao AngularJS) i Angular 2.
Radni okvir Angular 1 je postao jedan od popularnijih JavaScript radnih okvira za razvoj veb aplikacija. Njegovi nedostaci su postali previše istaknuti čim bi aplikacija postala veća i složenija.

Umesto ažuriranja verzije Angular 1, nastao je Angular 2 koji je kompletna prerada verzije Angular 1, predstavljajući potpuno novu arhitekturu aplikacije izgrađenu od nule u programskom jeziku TypeScript. TypeScript je nadskup programskog jezika JavaScript kreiran od strane kompanije Microsoft.
Nastao je kao potreba da se nadogradi postojeći JavaScript i podržan je u modernim operativnim sistemima. Zbog nekih nedostataka programskog jezika JavaScript, TypeScript je implementiran kao njegov nadskup i kompilira se nazad u JavaScript. Svi postojeći JavaScript fajlovi su validni TypeScript fajlovi.
Najbitnija novina je uvođenje tipova podatka, kao i klasa i modula. Tako je dodavanjem ovih funkcija u JavaScript omogućeno da se aplikacije pišu u modernom objektno orijentisanom programskom jeziku.
Većina poznatih razvojnih okruženja imaju podršku za TypeScript (Visual studio, Eclipse, Intellij Idea, Web storm, Sublime Text, Atom, Visual studio Code itd.). Razlog zašto se preporučuje TypeScript za razvoj Angular aplikacija zasnovan je na činjenici da je sam Angular napisan na ovom jeziku.
Napomena:Više o samom jeziku TypeScript možete saznati u zasebnom kursu.

Angular 4 je unapređena verzija verzije Angular 2.
Napomena:Za učenje verzija Angular 2 i 4, nije potrebno predznanje verzije Angular 1.
Napomena:Angular 3 verzija je preskočena zbog konflikta između brojeva verzija.

Funkcije radnog okvira Angular 4 obuhvataju sve što je potrebno modernoj veb aplikaciji:
  • interoperabilnost komponente,
  • univerzalnu podršku za više platformi i uređaja,
  • fleksibilnost,
  • napredan mehanizam rutera sa podrškom za razdvajanje i komponentiranje definicija rute itd.

Veb komponente

Angular aplikacije su napravljene od komponenti. Komponenta se sastoji od HTML šablona i klase komponente, dizajniranih da budu upotrebljeni zajedno za razvoj elemenata funkcije sa višim nivoom vizuelne izražajnosti i mogućnošću ponovne upotrebe.
Napraviti direktorijum "src". U njemu će se čuvati svi fajlovi koji se tiču aplikacije.
Zatim kreirati direktorijum unutar direktorijuma "src" koji pod nazivom "app" i u njemu sačuvati datoteku "glavna.komponenta.ts". U datoteci "glavna.komponenta.ts" uneti sledeći kod: import { Component } from '@angular/core'; @Component({ selector: 'moja-aplikacija', template: `<h1>Zdravo svete!</h1>` }) export class GlavnaKomponenta { }
Napomena:Vrednost šablona nije pod znacima navodnika, već se koriste znaci za akcentovanje.
Predstavljen je primer najjednostavnije komponente koja prikazuje tekst na ekranu "Zdravo svete!".
U prvom delu koda je uvezen najosnovniji tip i funkcija koja će biti potrebna za razvoj osnovne komponente.
Svaka komponenta sadrži dekorator @Component funkciju, koja je uvezena iz biblioteke @angular/core.
Postoji nekoliko vrsta dekoratora, a svi su prepoznatljivi po simbolu @, koji se upotrebljava kao prefiks.
U primeru ukazujemo kompilatoru da je klasa GlavnaKomponenta, u stvari, Angular komponenta.
Klasa kontrolera koja je kreirana pruža sve što je potrebno za instanciranje objekta.
U dekoratoru se nalazi selektor "moja-aplikacija", a svojstvo šablona definiše unutrašnju HTML strukturu napravljene komponente. Prilagođeni elementi koji kapsuliraju HTML šablone su osnova veb komponenata.
Element moja-aplikacija se nalazi u datoteci "index.html" i pozivom aplikacije se pomoću ovog elementa poziva kreirana veb komponenta, tj. "index.html" je stranica koja se prikazuje u pretraživaču. <moja-aplikacija>Učitava se aplikacija...</moja-aplikacija> Kreirati HTML datoteku u osnovnom direktorijumu radnog prostora pod nazivom "index.html", a zatim u nju dodati kod koji sadrži Angular 4 biblioteke koje su potrebne za razvoj Angular aplikacije <!DOCTYPE html> <html> <head> <title>Moja prva Angular aplikacija</title> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1"> <base href="/"> <link rel="stylesheet" href="styles.css"> <!-- Polyfill(s) for older browsers --> <script src="node_modules/core-js/client/shim.min.js"></script> <script src="node_modules/zone.js/dist/zone.js"></script> <script src="node_modules/systemjs/dist/system.src.js"></script> <script src="systemjs.config.js"></script> <script> System.import('main.js').catch(function(err){ console.error(err); }); </script> </head> <body> <base href="/"> <moja-aplikacija>Učitava se aplikacija...</moja-aplikacija> </body> </html> Element moja-aplikacija u datoteci "index.html" nije običan element. To je Angular komponenta sa definisanim selektorom moja-aplikacija.

Korenski (glavni) modul

Angular aplikacije su modularne. Svaka Angular aplikacija ima bar jedan modul, tzv. korenski (glavni) modul.
On služi za razvoj aplikacije, deklarišući komponente. Za sada on deklariše samo komponentu GlavnaKomponenta. import { NgModule } from '@angular/core'; import { BrowserModule } from '@angular/platform-browser'; @NgModule({ imports: [ BrowserModule ], providers: [ ], declarations: [ GlavnaKomponenta ], exports: [ GlavnaKomponenta ], bootstrap: [ GlavnaKomponenta ] }) export class AppModule { } NgModule je dekorator (uvozimo ga iz biblioteke @angular/core), koji vraća metadata objekat čije osobine opisuju modul. Najvažnije osobine su:
  • imports - drugi moduli čije izvezene klase su potrebne u šablonima komponenata u ovom modulu
  • providers - servisi koji postaju dostupni svim komponentama u aplikaciji
  • declarations - prikazane klase koje pripadaju ovom modulu. Angular sadrži tri tipa takvih klasa: komponente (eng. components), direktive (eng. directives) i filtere (eng. pipes)
  • exports - podskup deklaracija koje bi trebale da budu vidljive u šablonima komponenata u drugim modulima
  • bootstrap - instancira klasu kontrolera koja se prosleđuje kao argument i koristi je za konstrukciju cele aplikacije
Aplikacija se pokreće tako što u trećoj TypeScript datoteci (sačuvati je u direktorijumu "src" pod nazivom "main.ts") bi se uneo sledeći kod koji se kompilira i pokreće u pretraživaču. import { platformBrowserDynamic } from '@angular/platform-browser-dynamic'; import { AppModule } from './app/app.module'; platformBrowserDynamic().bootstrapModule(AppModule); JavaScript, takođe, ima module za upravljanje JavaScript objektima. Oni su potpuno različiti od Angular modula.
Štaviše, Angular je skup JavaScript modula (u daljem tekstu ćemo ih nazivati bibliotekama) o kojima je bilo reči u uvodu.
Ime svake Angular biblioteke počinje prefiksom @angular. Instaliraju se pomoću npm paketa, a u aplikaciju se uvoze pomoću ključne reči import.
U programskom jeziku JavaScript svaka datoteka je modul i svi objekti definisani u toj datoteci pripadaju tom modulu.
