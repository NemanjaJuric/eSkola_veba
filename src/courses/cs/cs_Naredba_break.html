Naredbe za promene toka izvršavanja petlji

Postoje dve ugrađene naredbe koje mogu da promene tok izvršavanja petlji i mogu se koristiti u kombinaciji sa bilo kojom od navedenih naredbi ponavljanja.
Prva je naredba break, koja prekida proces izvršavanja petlje, a druga je naredba continue koja prekida izvršavanje jedne iteracije petlje.

Naredba Break


Način na koji naredba break funkcioniše ako se pozove u petlji može se ilustovati na sledeći način:





Primer1. Obezbediti korisniku unosenje prirodnog broja, sve dok ne unese prirodan broj deljiv sa 3, nakon kojeg se ispisuje rezultat deljenja.
using System; namespace ConsoleApp3 { class Program { static void Main(string[] args) { while (true) { int n; Console.WriteLine("Unesite broj: "); n = int.Parse(Console.ReadLine()); int broj = 0; if (n % 3 == 0) { broj = n / 3; Console.WriteLine("Rezultat je " + broj); break; } } } } }
Rezultat je
"Unesite broj :"
4
"Unesite broj :"
8
"Unesite broj :"
9
"Rezultat je 3"

Ukoliko unesemo broj 4, proverom u if naredbi dobijamo da on nije deljiv sa 3, pa ce od nas biti zahtevano ponovno unosenje broja, sve dok ne unesemo broj deljiv sa 3.
Izabrali smo da napravimo beskonačnu petlju (uslov za naredbu while je uvek tačan), ali čim dobijemo odgovarajući broj izlazimo iz petlje naredbom break
i program nastavlja sa naredbom posle while-a, odnosno ispisuje rezultat deljenja unetog broja sa 3.
