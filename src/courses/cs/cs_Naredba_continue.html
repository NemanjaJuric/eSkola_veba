Naredba Continue


Naredba continue ne prekida celu petlju već trenutnu iteraciju i može se ilustrovati na sledeći način



Kada se u toku izvršavanje petlje u nekoj grani programa naiđe na naredbu continue, prekida se izvršavanje trenutne iteracije, odnosno preskaču se naredbe u bloku naredbi 2
i tok izvršavanja programa vraća se na ispitivanje uslova za sledeću iteraciju petlje.

Primer1. Omoguciti korisniku unos 5 celih brojeva. Izracunati zbir samo pozitivnih brojeva.
using System; namespace ConsoleApp3 { class Program { static void Main(string[] args) { int i; int zbir = 0; for (i = 1;i<=5;i++) { int n; Console.WriteLine("Unesite broj: "); n = int.Parse(Console.ReadLine()); if (n < 0) { continue; } zbir += n; } Console.WriteLine("Zbir = " + zbir); } } }




Rezultat je
"Unesite broj :"
4
"Unesite broj :"
-3
"Unesite broj :"
2
"Unesite broj :"
-2
"Unesite broj :"
8
"Zbir = 14".

U svakoj iteraciji učitavamo broj preko konzole i ako je unet negativan broj (uslov u naredbi if će biti ispunjen za brojeve -2, -3), naredbom continue prelazimo na sledeću iteraciju,
odnosno preskačemo naredbu koja sabira uneti broj sa trenutnom sumom.
