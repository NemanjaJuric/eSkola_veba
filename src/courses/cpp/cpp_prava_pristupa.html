Prava pristupa


Zaštita podataka je jedna od važnih karakteristika objektno orijentisanog programiranja koja omogućava sprečavanje funkcija programa da direktno pristupe podacima jedne klase.

Modifikatori pristupa koriste se za postavljanje granica dostupnosti članova klase, bilo da se radi o atributima ili metodama. Možemo koristiti jedan, dva ili više modifikatora pristupa u istoj klasi da bismo postavili različite granice za različite članove klase.

Modifikatori pristupa u programskom jeziku C++ definišu pravila kontrole pristupa. C++ poseduje 3 ključne reči:
  • public - javno
  • private - privatno
  • protected - zaštićeno
Pravo pristupa Klasa Nasleđena klasa main()
Public + + +
Private + - -
Protected + + -

Public pristup

Public pristup podrazumeva da će svi članovi klase koji su proglašeni javnim biti dostupni svima. Članovima podataka i metodama članova koji su public mogu pristupiti i druge klase. Otuda postoje i šanse da bi ih mogli promeniti. Dakle, ključni članovi jedne klase ne smeju biti public.

Primer class Osoba { public: string imePrezime; string pol; double mesecnaZarada; void predstaviSe(); };

Private pristup

Private ključna reč podrazumeva da niko ne može pristupiti članovima klase koji su proglašeni kao privatni, izvan te klase. Ako neko pokuša da pristupi privatnim članovima klase, dobiće grešku u vreme kompajliranja. Po default-u promenljive klase i metode su privatne.

Primer class Osoba { private: string imePrezime; string pol; double mesecnaZarada; void predstaviSe(); };

Protected pristup

Poslednji, protected pristup je sličan private pristupu. Naime, on podrazumeva nepristupačnost podacima klase izvan te klase, ali istim se može pristupiti iz bilo koje potklase te klase (ako klasu A nasleđuje klasa B, tada je klasa B potklasa klase A. O nasleđivanju će biti više reči u nastavku).

Primer class Osoba { protected: string imePrezime; string pol; double mesecnaZarada; void predstaviSe(); };
