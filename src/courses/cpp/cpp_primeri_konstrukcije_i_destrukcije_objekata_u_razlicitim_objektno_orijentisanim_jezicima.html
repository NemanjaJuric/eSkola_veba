Primeri konstrukcije i destrukcije objekata u različitim objektno orijentisanim programskim jezicima


Između objektno orijentisanih programskih jezika postoji dosta sličnosti.
Daćemo primer između Jave i C++-a. Oba su objektno orijentisana, podržavaju apstrakciju, klase, objekte i slično. I Java i C++ podržavaju konstruktore, međutim način na koji se oni pozivaju su drugačiji.

Na primer, u Javi se konstruktor mora pozvati pomoću operatora new(). Ne postoji drugi način da se eksplicitno pozove konstruktor tokom kreiranja objekta, dok se u C++-u to može učiniti bez operatora new.
Kao što je već rečeno u prethodnoj lekciji, C++ ima i konstruktor i destruktuor, dok Java nema destruktor, već samo konstruktor.

Neke razike ćemo sada navesti:

Deklaracija klase:

C++: class mojaKlasa{};
Java: class mojaKlasa{}

Deklaracija metoda:

Ista, osim što u Javi uvek moraju biti deo klase i mogu imati prefiks public, protected, private.

Konstruktor i destruktor:

Konstruktor i u Javi i u C++-u ima isto ime kao i klasa. Java nema destruktor.

Zajednički članovi klase:

Isto kao u deklaraciji metoda, osim što Java pruža statičke inicijalizovane blokove za inicijalizaciju statičkih promenljivih.
Java:
class mojaKlasa {
static private int x;
{ x = 5; } // staticki inicijalizovani blok
}

Deklaracija objekta:

C++:
mojaKlasa x; //objekat alociran na steku
mojaKlasa *x = new mojaKlasa; //objekat alociran na hipu
Java:
mojaKlasa x = new mojaKlasa(); //objekat je uvek alociran na hipu

Pristup objektu:

C++:
//ako je objekat alociran na steku, metodu pristupamo pomocu tacke
mojaKlasa x;
x.moj_metod;

//-> za pristup poljima klase kada radimo sa pokazivacima
mojaKlasa *x = new mojaKlasa();
x->moj_metod;

Java:
//uvek se radi sa pokazivacima
mojaKlasa x = new mojaKlasa();
x.moj_metod;
