Instanciranje klase


Instanciranje klase je stvaranje primerka (instance) objekta na jeziku objektno orijentisanog programiranja. Instanciranom objektu se dodeljuje ime i prostor u memoriji ili na disku koristeći strukturu opisanu u deklaraciji klase.

U programskom jeziku C++ instanciranje klase znači stvaranje objekta, dok u programskom jeziku Java instanciranje klase stvara određenu klasu. Rezultati u oba jezika su isti (izvršne datoteke), tako da nema suštinske razlike u upotrebi, već samo u terminologiji.

Operator new

Instanciranje se vrši pomoću ključne reči new.
Operator new je operator koji označava zahtev za dodelu memorije na hipu. Ako je na raspolaganju dovoljno memorije, operator new inicijalizuje memoriju i vraća adresu novo-dodeljene i inicijalizovane memorije na promenljivu pokazivača.

Operator new instancira klasu dodeljivanjem memorije za novi objekat tog tipa i zahteva jedan argument, a to je poziv konstruktora.
Konstruktor predstavlja posebnu metodu odgovornu za inicijalizaciju novih objekata određenog tipa. Operator new kreira objekat, dok ga konstruktor inicijalizuje. Više o konstruktoru će biti u nastavku.

Operator new vraća referencu na novostvoreni objekat. Ova referenca se može dodeliti promenljivoj odgovarajućeg tipa.

Primer instanciranja klase u programskom jeziku C++ Osoba* osoba1 = new Osoba("Milica Jovanovic", "z", "23.08.1998.", "Kraljevo");
Primer instanciranja klase u programskom jeziku Java Osoba osoba1 = new Osoba("Milica Jovanovic", "z", "23.08.1998.", "Kraljevo");

Razlika između objekta i instance

U objektno orijentisanim jezicima se objekat i instanca često smatraju istim stvarima, ali to ponekad može zbuniti početnike koji se prvi put susreću sa ovim pojmovima.
Da li su objekat i instanca iste stvari ili se razlikuju? Zašto ponekad koristimo objekat, a ponekad instancu?

Jedan od razloga korišćenja termina instanca može biti zato što su instance zapravo predstavljanje klase u određenom trenutku.

Dakle, objekat predstavlja tip koji definiše klasu koja sadrži podatke i metode za manipulaciju tim podacima. Instanca objekta je zapravo realizacija tipa kog objekat predstavlja. Program može imati nula, jednu ili više instanci objekata koje definiše.

Ukoliko posmatramo iz drugog ugla, objekat predstavlja neprekidni blok memorije koji sadrži stvarne podatke, koji ovaj objekat razlikuju od drugih objekata, dok instanca predstavlja memorijski blok koji sadrži referencu na objekat. Drugim rečima, instance će sadržati adresu početnog bloka memorije u kojem se objekat nalazi.

Zapravo se ime instance može koristiti za pristup početku delu memorije gde je smešten objekat.
Životni vek objekta i instance nisu povezani. Jednom kada se unište sve instance koje pokazuju na objekat, on će biti uništen.

Primer u programskom jeziku C++ Osoba osoba2; značenje: kreiran je objekat tipa Osoba i možemo mu pristupiti preko instance čiji je naziv osoba2. Osoba* osoba3 = new Osoba("Petar Krstic", "m", "03.10.1993.", "Ruma"); značenje: dinamički je kreirana instanca objekta klase Osoba pod nazivom osoba3, preko koje mu možemo pristupiti.



Operator delete

Kod dinamičkog kreiranja instance objekta klase, potrebno je izvršiti i dealokaciju (oslobađanje memorije), koja se vrši pomoću ključne reči delete.
Operator delete se može definisati na osnovu klase: ukoliko ne postoji definicija operatora za datu klasu, poziva se globalni operator delete, a ukoliko se delete poziva za oslobađanje objekta klase koja ima destruktor, oslobađanje se vrši putem destruktora, o kome će u nastavku biti više reči.

Primer primene operatora delete Osoba* osoba3 = new Osoba("Petar Krstic", "m", "03.10.1993.", "Ruma"); ... delete osoba3;
