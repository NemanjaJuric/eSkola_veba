Konstrukcija i destrukcija objekta

Konstruktor

Konstruktor je specijalna funkcija koja izvršava inicijalizaciju svakog objekta. Kompajler poziva konstruktor kad god se kreira objekat. Konstruktori inicijalizuju vrednosti članovima objekta nakon što je objektu dodeljena memorija.

Po čemu se konstruktori razlikuju od obične funkcije?
  • konstruktor ima isto ime kao i sama klasa,
  • konstruktor nema povratnu vrednost, dok funkcija u većini slučajeva ima,
  • konstruktor se automatski poziva kada se kreira objekat, dok funkciju treba pozvati eksplicitno pomoću objekta klase,
  • kada ne kreiramo nijedan konstruktor u klasi, C++ kompajler generiše podrazumevani konstruktor za nas (ne očekuje parametre i ima prazno telo), dok kod funkcije to nije slučaj.

Definicija konstruktora

Konstruktor može biti implementiran unutar klase ili eksterno van deklaracije klase.

Primer definicije konstruktora unutar klase class Trougao { private: int stranicaA; int stranicaB; int stranicaC; public: Trougao() { //implementacija konstruktora } }; Varijanta koja omogućava definiciju konstruktora izvan deklaracije klase class Trougao { private: int stranicaA; int stranicaB; int stranicaC; public: Trougao(); //deklaracija konstruktora }; Trougao::Trougao() { //implementacija konstruktora }
Naziv za :: je operator rezolucije dosega. Na primer: Trougao::stranicaA se odnosi na promenljivu stranicaA koja je deklarisana unutar dosega klase Trougao.

U klasi može biti više različitih konstruktora, o čemu će biti više reči u nastavku.

Destruktor

S druge strane, destruktor se koristi za uništavanje objekta klase. Destruktor se primenjuje kada treba osloboditi (dealocirati) memoriju koju je konstruktor zauzeo (alocirao) ili neke druge resurse.

Osobine destruktora:
  • destruktor se poziva kada je objekat uništen, kada izađe van oblasti važenja klase ili kada je obrisan pomoću operatora delete,
  • destruktor se ne može deklarisati kao static ili const,
  • destruktor nema argumente,
  • destruktor nema povratni tip (čak ni void tip),
  • objekat i destruktor ne mogu biti članovi iste unije,
  • destruktor treba deklarisati sa pravom pristupa public,
  • programer ne može pristupiti adresi destruktora.

Destruktor se poziva automatski u sledećim situacijama:
  • kada se funkcija završava,
  • kada se program završava,
  • kada se završava blok koji sadrži lokalne promenljive,
  • kada se poziva operator brisanja.

Primer definicije destruktora unutar klase class Trougao { private: int stranicaA; int stranicaB; int stranicaC; public: ~Trougao() { //implementacija destruktora } }; Varijanta koja omogućava definiciju destruktora izvan deklaracije klase class Trougao { private: int stranicaA; int stranicaB; int stranicaC; public: ~Trougao(); //deklaracija destruktora }; Trougao::~Trougao() { //implementacija destruktora } Ukoliko ne definišemo destruktor, kompajler će nam omogućiti podrazumevani.
Podrazumevani destruktor dobro funkcioniše ukoliko u klasi nismo dinamički dodelili memoriju ili pokazivač. U suprotnom, trebalo bi napisati destruktor za oslobađanje memorije pre nego što instanca bude uništena. To se mora učiniti kako bi se izbeglo tzv. curenje memorije.

Može li u klasi postojati više destruktora?
Ne! Može postojati samo jedan destruktor u klasi sa imenom klase kojoj prethodi ~, bez parametara i bez povratnog tipa.
