Rezime


Program može pomoću izuzetaka nagovestiti da je došlo do greške. Da biste bacili/ispalili izuzetak, koristite throw naredbu i obezbedite objekat izuzetka - potomak Throwable klase - kako biste dosatvili informaciju o konkretnoj grešci koja se dogodila. Metoda koja baca/ispaljuje ne obrađeni, provereni izuzetak mora u svojoj deklaraciji sadržati throws klauzulu.

Program može uhvatiti izuzetke kombinujući try blok sa catch i finally blokovima.

  • try blok identifikuje blok koda u kojem može doći do izuzetka.
  • catch blok identifikuje blok koda, poznat kao rukovalac izuzecima, koji može obraditi određeni tip izuzetka.
  • finally blok identifikuje blok koda čije se izvršavanje garantuje nakon koda obuhvaćenog try blokom i pravo je mesto za zatvaranje datoteke, oporavak resursa i sva ostala čišćenja.

try naredba treba da sadrži najmanje jedan catch ili finally blok i može imati više catch blokova.

Klasa objekta izuzetka označava tip bačenog/ispaljenog izuzetka. Objekat izuzetka može sadržati dodatne informacije o grešci, uključujući poruku o grešci. Sa lančanim izuzecima, izuzetak može ukazivati na izuzetak koji ga je prouzrokovao, a taj može ukazivati na izuzetak koji je njega prouzrokovao i tako dalje.
